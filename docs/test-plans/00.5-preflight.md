# Test plan — Preflight (Plan 00.5)

Manual end-to-end verification that the preflight foundations work. Each step
ends in a checkbox you can tick once you see the expected result.

Run everything through Docker:

```bash
docker compose up -d db app
docker compose logs -f app   # watch live
```

Prereqs:
- Docker Desktop running
- Port 3000 free
- `.env` populated (defaults from `.env.example` are fine for dev)

---

## 1. Stack starts cleanly

- [ ] `docker compose up -d db app` brings both containers up healthy
- [ ] `docker compose logs app` shows:
  - `[JobsService] pg-boss started`
  - `[NestApplication] Nest application successfully started`
  - `Cash Van API listening on http://0.0.0.0:3000`
- [ ] No `Cannot find module` or compile errors in the log tail

---

## 2. Migrations applied (RLS infrastructure)

```bash
docker compose run --rm app npm run migration:show
```

- [ ] Output includes `[X] RlsInfrastructure1715700000000` (X = applied)

Verify the SQL helpers exist:

```bash
docker compose exec db psql -U cashvan -d cashvan -c \
  "SELECT proname FROM pg_proc WHERE proname LIKE 'enable_tenant_rls%';"
```

- [ ] Returns two rows: `enable_tenant_rls` and `enable_tenant_rls_on`

---

## 3. pg-boss schema provisioned

```bash
docker compose exec db psql -U cashvan -d cashvan -c \
  "SELECT count(*) AS pgboss_tables FROM information_schema.tables WHERE table_schema='pgboss';"
```

- [ ] `pgboss_tables` >= 5

---

## 4. Health endpoint (Public + SkipTenantContext)

```bash
curl -s http://localhost:3000/api/v1/health | jq
```

Expected:

```json
{
  "success": true,
  "data": { "status": "ok", "db": "up" },
  "timestamp": "<iso ts>"
}
```

- [ ] HTTP 200
- [ ] `success === true`
- [ ] `data.db === "up"`
- [ ] Response uses the shared envelope (no `data` field is wrapped twice)

---

## 5. Error envelope on an unknown route

```bash
curl -s -o /dev/null -w "%{http_code}\n" http://localhost:3000/api/v1/no-such-endpoint
curl -s http://localhost:3000/api/v1/no-such-endpoint | jq
```

Expected:

```json
{
  "statusCode": 404,
  "message": "Cannot GET /api/v1/no-such-endpoint",
  "error": "Not Found",
  "path": "/api/v1/no-such-endpoint",
  "timestamp": "<iso ts>"
}
```

- [ ] HTTP 404
- [ ] Response shape matches the documented error envelope exactly

---

## 6. Login still works (JWT v: 2)

Use a seeded user (or seed first with `docker compose run --rm app npm run seed`).

```bash
curl -s -X POST http://localhost:3000/api/v1/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"userNumber":"<seeded>","password":"<seeded>"}' | jq
```

Decode the returned `accessToken` (e.g. paste at jwt.io):

- [ ] HTTP 200
- [ ] Payload contains `"v": 2`
- [ ] Payload contains `sub`, `userNumber`, `userType`, `permissions`
- [ ] `tenantId` is undefined (plan 01 will populate it)

---

## 7. Authenticated request with no tenant context (v: 1 / v: 2 without tenantId)

Until plan 01, every token is effectively tenant-less. Any tenant-scoped query
still works because the `TenantTxInterceptor` short-circuits when CLS has no
tenant.

```bash
TOKEN="<accessToken from step 6>"
curl -s -H "Authorization: Bearer $TOKEN" http://localhost:3000/api/v1/users | jq '.success'
```

- [ ] `true` — request succeeds (no tenant scoping yet)

---

## 8. Unit tests pass

```bash
docker compose run --rm --no-deps app npm test -- --testPathPattern='currency|geo'
```

- [ ] `Test Suites: 2 passed, 2 total`
- [ ] `Tests: 19 passed, 19 total`

## 9. Storage volume is mounted

```bash
docker compose exec app sh -c "ls -la /app/storage"
```

- [ ] Directory exists (created by the volume mount)
- [ ] Writable by the node user (you can `touch /app/storage/probe` from inside)

## 10. Skipping the tenant-tx interceptor

Hit `/api/v1/health` again with `docker compose logs app -f` open in another
terminal:

```bash
curl -s http://localhost:3000/api/v1/health > /dev/null
```

- [ ] No `BEGIN; SET LOCAL app.tenant_id ...; COMMIT;` block appears in the
      query log (with `DB_LOGGING=true`) — confirms `@SkipTenantContext()`
      is honored
