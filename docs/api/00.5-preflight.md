# API — Preflight (Plan 00.5)

Preflight adds no new endpoints. It establishes the foundations every later
feature depends on: shared envelopes, request context, decorators, RLS plumbing.

This doc records the conventions every feature MD will reference.
 
---

## Base URL

All routes live under `/api/v1/`.

## Authentication

Bearer JWT in `Authorization: Authorization: Bearer <token>` header. Login at
`POST /api/v1/auth/login` (already exists; preflight only adds `v: 2` to the
issued payload).

JWT payload (v: 2):

```json
{
  "sub":        "<user uuid>",
  "v":          2,
  "userNumber": "U001",
  "userType":   "ADMIN" | "MANAGER" | "SALES" | "DRIVER",
  "permissions": { "canMakeVoucher": true, "...": "..." },
  "tenantId":   "<tenant uuid>",   // populated once plan 01 ships
  "role":       "admin"             // alias of userType for new RBAC
}
```

Tokens with `v > 2` are rejected (`401 token_outdated`). Older `v: 1` tokens
without `tenantId` are accepted but tenant-scoped queries will return zero
rows; clients should re-login on first 401.

## Shared response envelopes

These shapes are emitted by `TransformInterceptor` and `HttpExceptionFilter`
respectively. **Every endpoint uses them — do not redocument them per route.**

### Success envelope
```json
{
  "success": true,
  "data": "<T — per-endpoint payload>",
  "timestamp": "2026-05-18T09:18:21.633Z"
}
```

### Error envelope (identical across every endpoint)
```json
{
  "statusCode": 409,
  "message": "Duplicate value violates unique constraint",
  "error": "ConflictError",
  "path": "/api/v1/reps",
  "timestamp": "2026-05-18T09:18:21.633Z"
}
```

Per-endpoint docs only list the **possible** `<status> <errorName>` pairs:

| Status | error |
|---|---|
| 400 | `BadRequestError`, `BadRequestException` |
| 401 | `UnauthorizedException` |
| 403 | `ForbiddenException` |
| 404 | `NotFoundException` |
| 409 | `ConflictError` |
| 422 | `BadRequestException` (validation pipe) |
| 429 | `ThrottlerException` |
| 500 | `InternalServerError`, `QueryFailedError` |

## Decorators introduced by preflight

| Decorator | Purpose |
|---|---|
| `@Public()` (already existed) | Skip JWT guard entirely. Use for login, register, public webhooks. |
| `@SkipTenantContext()` *(new)* | Skip the tenant-tx interceptor. Use for routes that legitimately have no tenant in CLS (health, login). Authenticated cross-tenant admin routes should also use this and apply an explicit role check. |

Apply both on routes that are public AND tenant-less (e.g. login):

```ts
@Public()
@SkipTenantContext()
@Post('login')
```

## Tenant isolation (Row-Level Security)

Every authenticated request that is NOT marked `@SkipTenantContext()`:

1. Goes through `JwtAuthGuard` → `JwtStrategy.validate()` writes
   `{ tenantId, userId, role }` into CLS.
2. Hits `TenantTxInterceptor` → opens a TypeORM transaction and runs
   `SET LOCAL app.tenant_id = '<uuid>'` as the first statement.
3. Postgres RLS policies on `tenant_id`-scoped tables filter every query
   automatically.
4. On commit/rollback the SET LOCAL evaporates — the connection returns to
   the pool clean.

Migrations use a superuser role that bypasses RLS. Application connections
do not.

To enable RLS on a new table in a feature migration:

```sql
SELECT enable_tenant_rls('<table_name>');
-- or, if the tenant-scoping column is named something other than tenant_id:
SELECT enable_tenant_rls_on('<table_name>', '<col>');
```

## Currency

INTEGER **fils** (1 JOD = 1000 fils) is the only money type in the DB and the
service layer. JSON payloads also use fils. Convert at the JoFotara JSON
boundary only.

Helpers: `src/common/utils/currency.util.ts`

| Function | Behavior |
|---|---|
| `filsToJod(n)` | `1234 → '1.234'` |
| `jodToFils(s)` | `'1.234' → 1234` |
| `roundFils(n)` | Half-up integer rounding |

## Geometry

GeoJSON Polygons stored as `JSONB`. Validation + point-in-polygon happens in
Node (no PostGIS dependency).

Helpers: `src/common/geo/geo.util.ts`

| Function | Behavior |
|---|---|
| `validateGeoJsonPolygon(input)` | Throws on malformed input; returns typed `GeoJsonPolygon`. |
| `isPointInPolygon([lng,lat], polygon)` | Boolean. Inclusive on boundary. |

## Object storage

Provider-neutral interface: `STORAGE_SERVICE` (injection token).

Default adapter: local disk (`./storage/` volume-mounted). Swap with an S3
adapter via DI in production by re-providing `STORAGE_SERVICE`.

```ts
constructor(@Inject(STORAGE_SERVICE) private storage: StorageService) {}
await this.storage.put('cheques/abc.jpg', buffer, { contentType: 'image/jpeg' });
const url = await this.storage.getUrl('cheques/abc.jpg', 600);
```

---

## Cache

Provider-neutral interface: `CACHE_SERVICE` (injection token).

Default adapter: in-process LRU (`lru-cache`). Per-instance, not shared across
replicas. Used by AI services for response memoization. Do NOT cache
tenant-scoped DB rows here — let RLS handle that.

```ts
await this.cache.set(`ai:briefing:${tenantId}:${date}`, payload, 3600);
const cached = await this.cache.get<Briefing>(`ai:briefing:${tenantId}:${date}`);
```

## Background jobs

`JobsService` (Postgres-backed via `pg-boss`). No Redis dependency.

Tenant context is captured automatically at enqueue and re-established inside
CLS for the worker. Handlers can call tenant-scoped repositories normally.

```ts
// Enqueue (inside an HTTP request)
await this.jobs.enqueue('ai-cheque-extract', { chequeId });

// Register (in a module's onApplicationBootstrap)
await this.jobs.register<{ chequeId: string }>('ai-cheque-extract', async (data) => {
  // CLS is already scoped to the enqueuer's tenant.
  await this.processCheque(data.chequeId);
});
```

Disable for tests by setting `JOBS_ENABLED=false`.

## Health

`GET /api/v1/health` — `@Public() @SkipTenantContext()`, returns `{ status, db }`.

```bash
curl http://localhost:3000/api/v1/health
# {"success":true,"data":{"status":"ok","db":"up"},"timestamp":"..."}
```
