# Plan 00.5 — Preflight (foundations)

Depends on: nothing. **Must complete before plan 01.**

## Goal

Land the cross-cutting foundations every later plan assumes: multi-tenant request context, RLS plumbing, supporting infrastructure services, and global conventions. One PR.

## Decisions locked here

| Topic | Decision |
|---|---|
| Database | **Vanilla Postgres 15** — no PostGIS image swap |
| Multi-tenant isolation | DB-enforced RLS via `SET LOCAL app.tenant_id` inside a per-request transaction |
| Tx + CLS libraries | `typeorm-transactional` + `nestjs-cls` |
| Geometry | `JSONB` (GeoJSON), point-in-polygon via `@turf/boolean-point-in-polygon` |
| Money | `INTEGER` fils; conversion to JOD strings only at JoFotara boundary |
| Object storage | Local disk for dev (volume mount); S3-compatible adapter interface for prod |
| Queues | `pg-boss` (Postgres-backed) — avoids hard Redis dependency. Bull only if Redis is later added. |
| Caching | In-memory LRU (`lru-cache`) for v1; swappable cache interface for Redis later |
| Realtime | Socket.io, **single-instance** v1 (no Redis adapter); document scale-out as a future step |

- [ ] Helper function for plans to call:
  ```sql
  CREATE OR REPLACE FUNCTION enable_tenant_rls(tbl regclass) RETURNS void AS $$
  BEGIN
    EXECUTE format('ALTER TABLE %s ENABLE ROW LEVEL SECURITY', tbl);
    EXECUTE format($f$
      CREATE POLICY tenant_isolation ON %s
        USING (tenant_id = current_setting('app.tenant_id', true)::uuid)
        WITH CHECK (tenant_id = current_setting('app.tenant_id', true)::uuid)
    $f$, tbl);
  END;
  $$ LANGUAGE plpgsql;
  ```
